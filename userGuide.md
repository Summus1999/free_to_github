# Free to GitHub 用户指南

## 快速开始

1. 下载并安装 `Free to GitHub_2.0.0_x64-setup.exe`
2. 启动应用（自动请求管理员权限）
3. 点击「测速」→「启用加速」→「刷新DNS」

---

## 功能说明

| 按钮 | 功能 | 耗时 |
|------|------|------|
| 测速 | 测试所有 GitHub IP 延迟 | 10-30秒 |
| 启用加速/智能启用 | 使用最快 IP 配置 hosts | <1秒 |
| 禁用 | 恢复原始 hosts | <1秒 |
| 刷新DNS | 清除 DNS 缓存 | ~1秒 |
| Hosts文件 | 打开 hosts 文件夹 | 即时 |
| 打开GitHub | 跳转到 GitHub 官网 | 即时 |

---

## 测速结果说明

| 延迟 | 质量 | 颜色 |
|------|------|------|
| 0-50ms | 极佳 | 绿色 |
| 51-100ms | 良好 | 黄色 |
| 101-200ms | 一般 | 橙色 |
| >200ms | 较慢 | 红色 |

---

## 常见问题

### 启用后无法访问 GitHub？

1. 点击「刷新DNS」
2. 重启浏览器或清除浏览器缓存
3. 重新「测速」→「启用加速」

### 测速结果全红？

- 检查网络连接
- 尝试切换网络（WiFi/4G）
- 过段时间重试

### 用了一段时间变慢？

GitHub IP 可能变化，重新测速并启用即可。

### 如何卸载？

1. 点击「禁用」恢复 hosts
2. 在「设置」→「应用」中卸载「Free to GitHub」

---

## 最佳实践

- **首次使用**: 测速 → 启用加速 → 刷新DNS
- **定期优化**: 每月重新测速一次
- **网络切换后**: 重新测速
